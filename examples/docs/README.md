# zlog docs layer prototype

This example keeps the docs framework layer isolated from the core SSG engine. It uses normal zlog layouts, partials, static assets, and generated routes.

Included pieces:

- `layouts/partials/docs-header.shtml` provides the docs header and a search input annotated with `data-z-search-index="/search.json"`.
- `layouts/partials/docs-sidebar.shtml` provides a static sidebar for the guide and reference pages.
- `doc.shtml` renders `page.toc`, which is generated from Markdown headings.
- `static/docs.css` styles the docs shell without requiring runtime JavaScript.
- `search.json` is generated by `zlog build` and can be consumed by a future client-side search script.

Prototype boundaries:

- Sidebar support is implemented as reusable layout partials.
- Next and previous navigation is currently authored as normal Markdown links inside content pages.
- Table of contents rendering uses generated heading metadata from the core template binding.
- Callouts currently use Markdown blockquotes as a safe fallback until a dedicated callout syntax exists.
